# Markmap - Markdown 思维导图

Markmap 可以将 Markdown 文档自动转换为交互式的思维导图。本章将展示如何在 Markdown 中直接使用 Markmap 创建可视化的思维导图。

## 什么是 Markmap？

Markmap 是一个可视化工具，它可以：
- 将 Markdown 标题结构转换为思维导图
- 支持交互式展开/折叠节点
- 自动布局和美化
- 一目了然地查看文档层级

## 基本用法

在 Markdown 中使用 `markmap` 代码块即可创建思维导图：

````markdown
```markmap
# 根节点

## 第一个子节点

### 子子节点 1
### 子子节点 2

## 第二个子节点

### 另一个子节点
```
````

**效果展示：**

```markmap
# 根节点

## 第一个子节点

### 子子节点 1
### 子子节点 2

## 第二个子节点

### 另一个子节点
```

## 使用技巧

### 技巧 1：控制层级深度

建议保持在 3-4 层以内，过深会导致思维导图难以阅读。

✅ 好的做法：

````markdown
```markmap
# 主题
## 二级
### 三级
#### 四级（最多到这里）
```
````

❌ 不好的做法：

````markdown
```markmap
# 主题
## 二级
### 三级
#### 四级
##### 五级
###### 六级（太深了）
```
````

### 技巧 2：使用简洁的标题

✅ 好的做法：

````markdown
```markmap
## 用户管理
### 注册
### 登录
### 权限
```
````

❌ 不好的做法：

````markdown
```markmap
## 用户管理模块的详细说明
### 用户注册功能的实现方式
### 用户登录流程的详细步骤
```
````

### 技巧 3：保持结构平衡

尽量让每个分支的子节点数量相近，避免某个分支过于庞大。

### 技巧 4：快速预览文档结构

在编写长文档时，可以在文档开头添加一个 Markmap 来展示整体结构：

````markdown
# 完整文档标题

## 文档结构预览

```markmap
# 文档结构

## 第一部分
### 章节 1
### 章节 2

## 第二部分
### 章节 3
### 章节 4
```

## 第一部分

详细内容...
````
