# 表格

表格是展示结构化数据的最佳方式。Markdown 支持创建简单而优雅的表格。

## 基本表格语法

使用 `|` 分隔列，使用 `-` 创建表头分隔线。

### 最简单的表格

```markdown
| 列1 | 列2 | 列3 |
| --- | --- | --- |
| 数据1 | 数据2 | 数据3 |
| 数据4 | 数据5 | 数据6 |
```

效果：

| 列1 | 列2 | 列3 |
| --- | --- | --- |
| 数据1 | 数据2 | 数据3 |
| 数据4 | 数据5 | 数据6 |

### 简化写法

两端的 `|` 可以省略：

```markdown
列1 | 列2 | 列3
--- | --- | ---
数据1 | 数据2 | 数据3
数据4 | 数据5 | 数据6
```

效果相同：

列1 | 列2 | 列3
--- | --- | ---
数据1 | 数据2 | 数据3
数据4 | 数据5 | 数据6

## 对齐方式

使用冒号 `:` 控制列的对齐方式。

```markdown
| 左对齐 | 居中对齐 | 右对齐 |
| :--- | :---: | ---: |
| 左 | 中 | 右 |
| Left | Center | Right |
| 文本 | 文本 | 文本 |
```

效果：

| 左对齐 | 居中对齐 | 右对齐 |
| :--- | :---: | ---: |
| 左 | 中 | 右 |
| Left | Center | Right |
| 文本 | 文本 | 文本 |

**对齐规则：**
- `:---` 左对齐（默认）
- `:---:` 居中对齐
- `---:` 右对齐

## 复杂表格处理

### 长文本处理

对于包含长文本的单元格，可以使用 `<br>` 标签换行：

```markdown
| 问题 | 解决方案 |
| --- | --- |
| 表格列太宽 | 使用简短的文字<br>或者使用缩写 |
| 内容太多 | 分成多行显示<br>保持表格整洁 |
```

效果：

| 问题 | 解决方案 |
| --- | --- |
| 表格列太宽 | 使用简短的文字<br>或者使用缩写 |
| 内容太多 | 分成多行显示<br>保持表格整洁 |

### 使用 HTML 增强表格

标准 Markdown 表格不支持合并单元格，但可以使用 HTML：

```html
<table>
  <tr>
    <th>姓名</th>
    <th colspan="2">联系方式</th>
  </tr>
  <tr>
    <td>张三</td>
    <td>电话</td>
    <td>138xxxx1234</td>
  </tr>
  <tr>
    <td rowspan="2">李四</td>
    <td>电话</td>
    <td>139xxxx5678</td>
  </tr>
  <tr>
    <td>邮箱</td>
    <td>lisi@example.com</td>
  </tr>
</table>
```

## 最佳实践

### 1. 保持表格简洁

✅ 好的做法：
```markdown
| 名称 | 价格 |
| --- | ---: |
| 苹果 | ¥5 |
| 香蕉 | ¥3 |
```

❌ 不好的做法（列太多）：
```markdown
| 名称 | 价格 | 产地 | 重量 | 保质期 | 营养成分 | 储存方式 |
```

### 2. 使用合适的对齐方式

- 文本：左对齐
- 数字：右对齐
- 状态图标：居中对齐

### 3. 添加表头

始终为表格添加清晰的表头。

### 4. 考虑移动端显示

表格在小屏幕上可能显示不佳，考虑：
- 减少列数
- 使用缩写
- 必要时使用列表代替表格

### 5. 格式化表格代码

使用工具对齐表格的 `|` 符号，提高可读性：

```markdown
| 名称   | 价格  | 库存 |
| ------ | ----: | ---: |
| 苹果   | ¥5    | 100  |
| 香蕉   | ¥3    | 200  |
| 橙子   | ¥4    | 150  |
```
