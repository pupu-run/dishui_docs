# 列表和引用

## 无序列表（Unordered Lists）

无序列表使用 `-`、`*` 或 `+` 作为列表标记。

### 基本语法

```markdown
- 第一项
- 第二项
- 第三项

* 苹果
* 香蕉
* 橙子

+ 任务一
+ 任务二
+ 任务三
```

### 效果

- 第一项
- 第二项
- 第三项

* 苹果
* 香蕉
* 橙子

### 嵌套列表

使用缩进（2 或 4 个空格）创建嵌套列表。

```markdown
- 水果
  - 苹果
    - 红富士
    - 青苹果
  - 香蕉
  - 橙子
- 蔬菜
  - 白菜
  - 萝卜
```

效果：

- 水果
  - 苹果
    - 红富士
    - 青苹果
  - 香蕉
  - 橙子
- 蔬菜
  - 白菜
  - 萝卜

## 有序列表（Ordered Lists）

有序列表使用数字加 `.` 表示。

### 基本语法

```markdown
1. 第一步
2. 第二步
3. 第三步
```

效果：

1. 第一步
2. 第二步
3. 第三步

### 自动编号

Markdown 会自动处理编号，所以你可以都使用 `1.`：

```markdown
1. 第一项
1. 第二项
1. 第三项
```

效果仍然是：

1. 第一项
2. 第二项
3. 第三项

### 有序列表嵌套

```markdown
1. 准备工作
   1. 安装软件
   2. 配置环境
2. 开始编码
   1. 创建项目
   2. 编写代码
   3. 测试
3. 部署上线
```

效果：

1. 准备工作
   1. 安装软件
   2. 配置环境
2. 开始编码
   1. 创建项目
   2. 编写代码
   3. 测试
3. 部署上线

## 混合列表

可以混合使用有序和无序列表。

```markdown
1. 编程语言
   - Python
   - JavaScript
   - Go
2. 框架
   - Django
   - React
   - Gin
3. 工具
   - Git
   - Docker
   - VS Code
```

效果：

1. 编程语言
   - Python
   - JavaScript
   - Go
2. 框架
   - Django
   - React
   - Gin
3. 工具
   - Git
   - Docker
   - VS Code

## 任务列表（Task Lists）

GitHub Flavored Markdown 支持任务列表。

### 语法

```markdown
- [x] 完成第一章学习
- [x] 完成第二章学习
- [ ] 完成第三章学习
- [ ] 完成第四章学习
```

### 效果

- [x] 完成第一章学习
- [x] 完成第二章学习
- [ ] 完成第三章学习
- [ ] 完成第四章学习

### 嵌套任务列表

```markdown
- [x] 前端开发
  - [x] HTML
  - [x] CSS
  - [ ] JavaScript
- [ ] 后端开发
  - [ ] Python
  - [ ] 数据库
```

效果：

- [x] 前端开发
  - [x] HTML
  - [x] CSS
  - [ ] JavaScript
- [ ] 后端开发
  - [ ] Python
  - [ ] 数据库

## 引用块（Blockquotes）

使用 `>` 创建引用块。

### 基本语法

```markdown
> 这是一段引用文字。
```

效果：

> 这是一段引用文字。

### 多段引用

```markdown
> 第一段引用。
>
> 第二段引用。
```

效果：

> 第一段引用。
>
> 第二段引用。

### 嵌套引用

```markdown
> 第一层引用
>
> > 第二层引用
> >
> > > 第三层引用
```

效果：

> 第一层引用
>
> > 第二层引用
> >
> > > 第三层引用

### 引用中使用其他 Markdown 语法

```markdown
> ### 重要提示
>
> 这是一个 **非常重要** 的提示：
>
> - 记得保存文件
> - 记得备份数据
> - 记得测试代码
>
> *—— 来自经验丰富的开发者*
```

效果：

> ### 重要提示
>
> 这是一个 **非常重要** 的提示：
>
> - 记得保存文件
> - 记得备份数据
> - 记得测试代码
>
> *—— 来自经验丰富的开发者*

## 列表中的段落和引用

### 列表中的多段内容

```markdown
1. 第一项

   这是第一项的详细说明。需要缩进对齐。

   可以有多个段落。

2. 第二项

   > 这是一个引用
   
   这是第二项的说明。
```

效果：

1. 第一项

   这是第一项的详细说明。需要缩进对齐。

   可以有多个段落。

2. 第二项

   > 这是一个引用
   
   这是第二项的说明。

## 实践练习

创建一个项目计划文档，包含：

```markdown
# 项目开发计划

## 第一阶段：需求分析

> 重要：这个阶段需要充分沟通

1. 收集需求
   - 用户访谈
   - 竞品分析
   - 需求文档编写
2. 评审需求
   - [ ] 技术评审
   - [ ] 产品评审
   - [ ] 设计评审

## 第二阶段：开发实施

1. 前端开发
   - [x] 页面设计
   - [ ] 组件开发
   - [ ] 接口对接
2. 后端开发
   - [ ] 数据库设计
   - [ ] API 开发
   - [ ] 单元测试

## 第三阶段：测试上线

> 测试要覆盖所有功能点

- 功能测试
- 性能测试
- 安全测试
- 用户验收测试

---

**注意事项：**
- 每个阶段完成后需要评审
- 遇到问题及时沟通
- 保持文档更新
```
